# Лабораторная работа 3. Разработка синтаксического анализатора (парсера)
## Цель работы:
Изучить назначение и принципы работы синтаксического анализатора в структуре компилятора. Спроектировать грамматику, построить соответствующую схему метода анализа грамматики и выполнить программную реализацию парсера с нейтрализацией синтаксических ошибок методом Айронса. Интегрировать разработанный модуль в ранее созданный графический интерфейс языкового процессора.
## Студент: Осинцева Д.А. АП - 327  
## Постановка задачи

Разработать синтаксический анализатор (парсер) в соответствии с индивидуальным вариантом курсовой (расчетно-графической) работы, интегрировать его в приложение из лабораторной работы №1 и обеспечить наглядный вывод результатов анализа. 



Требования к разработке парсера:

Разработать грамматику для заданной синтаксической конструкции.

Построить схему метода анализа на основе разработанной грамматики.

Выполнить программную реализацию алгоритма работы синтаксического анализа.

Реализовать алгоритм нейтрализации синтаксических ошибок методом Айронса.

Входные данные — строка (текст программного кода) из области редактирования.

Выходные данные:

При успешном анализе корректной строки — сообщение об отсутствии ошибок.

При обнаружении ошибок — таблица с описанием каждой ошибки.

Требования к интеграции и интерфейсу:
    
Встроить парсер в ранее разработанный интерфейс (ЛР1) и связать его с кнопкой «Пуск» (или отдельной кнопкой для синтаксического анализа).

Окно вывода результатов должно содержать таблицу ошибок со следующими столбцами:

Неверный фрагмент (символ или фрагмент, вызвавший ошибку).

Местоположение (номер строки, позиция символа).

Описание ошибки (опционально).

В окне вывода также отображается общее количество найденных ошибок.

Реализовать навигацию по ошибкам: при щелчке на строке таблицы курсор в области редактирования должен устанавливаться на позицию ошибочного фрагмента.

## Перечень допустимых лексем:  
- **1** – Идентификатор 
- **2** – struct (ключевое слово)
- **3** – public (ключевое слово)
- **4** – string (ключевое слово)
- **5** – int (ключевое слово)
- **6** – float (ключевое слово)
- **7** – this (ключевое слово)
- **8** – Целое число 
- **9** – Число с плавающей точкой 
- **10** – Начало выражения  (`{`)
- **11** – Конец выражения (`}`)
- **12** – Конец оператора (`;`)
- **13** – Начало перечисления агрументов (`(`)
- **14** – Конец перечисления агрументов (`)`)
- **15** – Оператор присваивания (`=`)
- **16** – Синтаксический знак (`.`)
- **17** – Синтаксический знак (`,`)
- **18** – Синтаксический знак (`:`)
- **19** – Разделитель (пробел)
- **20** – Ошибка 
## Разработка грамматики

    <START> -> struct <ID>
     <ID> -> _id <OpCurlyBrace>
    <OpCurlyBrace> -> { <P> | <THIS> }
    <P> -> public <TYPE> | public _id <OpBrace>
    <TYPE> -> type_id <E>
    <E> -> ; <CloseCurlyBrace> | <P>
    <OpBrace> -> ( type_id <COMMA> | <CloseBrace>
    <COMMA> -> , <TYPE>
    <THIS> -> this <DOT>
    <DOT> -> id <EQUAL>
    <EQUAL> -> id <CloseCurlyBrace>
    <CloseBrace> -> } <OpCurlyBrace>
    <CloseCurlyBrace> -> } <CloseCurlyBrace> | }
    <IDENTIFIER> -> letter <IDENTIFIER_REM>
    <IDENTIFIER_REM> -> letter | digit | ε
    ‒ id = <IDENTIFIER>
    ‒ letter -> 'a' | 'b' | ... | 'z' | 'A' | 'B' | ... | 'Z'
    ‒ digit -> '0' | '1' | ... | '9'
    ‒ type -> ' int' | ' float' | ' string'
